16.1. Nadchodzące Programowania

Kilka tygodni (lub miesięcy) przed wydaniem nowej wersji Debiana, Menadżer Wydania wybiera nazwę kodową dla następnej wersji. Teraz, ponieważ wersja 8 jest już wydana, programiści są już zajęci pracą nad nową wersją, o nazwie kodowej Stretch…
Nie ma żadnej oficjalnej listy zaplanowanych zmian, a Debian nigdy nie czyni żadnych obietnic odnośnie celów technicznych nadchodzących wersji. Jednakże, można już zauważyć kilka programistycznych trendów, i możemy się założyć, że mogą się one urzeczywistnić (lub nie).
Większość pakietów, jeśli nie wszystkie, będzie robiona w sposób reprodukowalny, aby poprawić bezpieczeństwo i zaufanie. Oznacza to, że będzie możliwe odbudowanie — bit po bicie — identycznych pakietów binarnych z pakietów źródłowych, co w rezultacie pozwoli każdemu zweryfikować, że podczas budowania pakietu nie wydarzyły się żadne manipulacje.
W odnośnym temacie będzie podejmowanych wiele wysiłków, aby ulepszyć bezpieczeństwo, i aby osłabić zarówno „tradycyjne” ataki, jak i nowe zagrożenia, spowodowane masową inwigilacją.
Oczywiście, wszystkie główne pakiety oprogramowania będą miały tzw. duże wydanie (ang. major release). Najnowsze wersje różnych pulpitów (środowisk graficznych) będą miały lepszą użyteczność i funkcjonalność. Wayland, nowy serwer wyświetlania, który jest rozwijany jako bardziej nowoczesna, zastępcza alternatywa dla serwera X11, będzie dostępny (chociaż nie w sposób domyślny) dla niektórych najnowszych środowisk pulpitowych.
A new feature of the archive maintenance software, “bikesheds”, will allow developers to host special-purpose package repositories in addition to the main repositories; this will allow for personal package repositories, repositories for software not ready to go into the main archive, repositories for software that has only a very small audience, temporary repositories for testing new ideas, and so on.
